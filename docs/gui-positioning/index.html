# GUI Positioning This extension allows you to position your sprites in an interface-like manner. ## Positioning Sprites ![Blocks to position a sprite bound to a frame](https://orangeleaf36.is-a.dev/docs/gui-positioning/setspritepos.svg) With these blocks, you can position a sprite on a frame on an anchor. Anchors: - Top (y) - Bottom (y) - Left (x) - Right (x) - Center (both) ## Frames In this extension, "Frames" are equivalent to a `
` in HTML. You can position sprites that are bound to frames. The stage can also be used as a frame that a sprite can be bound to. ### Creating and Deleting Frames --- ![Blocks to create and delete frames](https://orangeleaf36.is-a.dev/docs/gui-positioning/createdeleteframe.svg) Since frames are not built into the project, they are created when the project is run instead. Creating an existing frame will overwrite it with the default size and scale instead of creating a new frame, so it is recommended you delete all of the frames beforehand. ### Positioning and Scaling --- ![Blocks to normally position and scale frames](https://orangeleaf36.is-a.dev/docs/gui-positioning/normalpositioningframe.svg) These are normal blocks to position and scale a frame. ![Blocks to position frames on an anchor](https://orangeleaf36.is-a.dev/docs/gui-positioning/guipositioningframe.svg) These are blocks that allows you to position frames on the stage the same way you position sprites bound to frames. ## Zoom (Camera Controls) ![enter image description here](https://orangeleaf36.is-a.dev/docs/gui-positioning/zoomblocks.svg) In the case that your project uses the zoom from Camera Controls, these blocks can scale the position and size of a frame to the zoom.
